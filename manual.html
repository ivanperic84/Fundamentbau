3

Standorte verwalten

Erstellen, bearbeiten, importieren

Neuen Standort anlegen

  1. + Kachel antippenIn der Kachelansicht die gestrichelte Kachel «Neuer Standort» antippen.
  2. Marker setzenRS-Marker und optional RKS- und BS-Marker auf der Karte positionieren.
  3. Felder ausfüllenKM, Mast-Nr., Gleis, Tagarbeit, Tiefe Soll, Massnahme und Zugang eintragen.
  4. SpeichernDer Standort erscheint sofort in der Übersicht.

Standort bearbeiten

In der Kachelansicht erscheinen beim Hover drei Buttons oben rechts:

✏ Bearbeiten

Öffnet die Erfassungsmaske mit den bestehenden Werten.

Tag zuweisen

Schnelles Zuweisen eines Tags direkt auf der Kachel.

Import aus Fundamentliste (Excel)

Über Einstellungen → Import kann eine SBB-Fundamentliste (.xlsx / .xlsm) importiert werden. Die App liest KM, Mast, Gleis, Massnahme, Fundamenttyp und Bodenkennwerte. RS/RKS/BS-Koordinaten werden beim Import nicht gesetzt — diese werden separat erfasst.

! Bei «OK» werden bestehende Daten ersetzt. «Abbrechen» fügt die importierten Standorte zu den bestehenden hinzu.

Mehrfachauswahl (Listenansicht)

In der Listenansicht können mehrere Standorte per Checkbox ausgewählt und gemeinsam bearbeitet werden: Status setzen, Tag zuweisen, Löschen.

8

Bauprojekt

Fundamenttypen, Bodenprofil, Statische Nachweise

In der Phase Bauprojekt werden die Fundamentplanung und die Baugrundparameter pro Standort festgelegt.

Massnahme

Neubau

Neues Fundament wird erstellt.

Abbruch & Neubau

Bestehendes Fundament wird abgebrochen und durch neues ersetzt.

Abbruch

Nur Abbruch des Bestandes ohne Neubau.

Bestand erhalten

Kein Eingriff — Standort wird in Bauprogramm ausgeblendet.

Fundamenttyp

Dem Standort wird ein Fundamenttyp-Profil aus der zentralen FT-Bibliothek zugewiesen (siehe Kapitel 12). Die Zuweisung erfolgt über das Dropdown oder per Kachelauswahl im Tab Fundamente.

Baugrundprofil

Das Baugrundprofil enthält die geotechnischen Kennwerte (USCS, ME-Wert, Reibungswinkel, Raumlast, Kohäsion, Grundwasserstand). Profile können aus der Fundamentliste importiert oder manuell angelegt werden und stehen projektübergreifend zur Verfügung.

Statische Nachweise

Im Tab Statische Nachweise können Berechnungsdokumente (Typ, Datum, Bearbeiter, Link) erfasst und verwaltet werden. Pro Standort sind mehrere Nachweise möglich.

Naturschutz

Felder zur Erfassung von Naturschutzauflagen: Lebensraum, Massnahmen, Auflagen. Relevant für den Umweltnachweis.

9

Baugrundphase

RS / RKS / BS Felddaten erfassen

In der Phase Baugrund werden die Felddaten der Sondierungen (RS, RKS, BS) erfasst. Jede Sondierungsart hat eigene Felder und kann unabhängig aktiviert oder deaktiviert werden.

Sondierungstypen

KürzelBezeichnungSpezifische Felder
RSRammsondierungTiefe Ist, Abbruchgrund, Grundwasser, Bemerkung
RKSRammkernsondierungTiefe Ist, Abbruchgrund, Kerngewinn, Grundwasser, Schichtbeschrieb
BSBaggerschlitz / BodensondierungTiefe, Befund, Bemerkung

Abbruchgründe

Endtiefe erreicht · Hindernis · Fels · Geräteabbruch · Zugang · Sonstiges

Koordinaten RS / RKS / BS

Die Standortkoordinaten (LV95) werden separat über den jeweiligen Dateiimport oder per manueller Eingabe erfasst — nicht über die Fundamentliste. Auf der Detailkarte werden die gesetzten Marker (RS weiss, RKS rot, BS gelb) angezeigt.

Baugrund-Tab (Übersicht)

Im Tab Baugrund der Übersicht sind alle Standorte tabellarisch mit ihren Felddaten aufgelistet. Export als Excel über die Export-Funktion möglich.

Ausführungsplanung

Schicht-Zuweisung und Betriebsplanung

In der Phase Ausführung wird jedem Standort ein Baupaket und eine Schicht-Nummer zugewiesen. Die App berechnet daraus automatisch das Ausführungsdatum.

Schicht-Zuweisung

FeldBeschreibung
BaupaketZuordnung zum Baupaket (z.B. «Block / Brunnen / Fels»)
Schicht-Nr.Nummer der Nacht/Schicht innerhalb des Pakets (1 = erste Nacht)
DatumWird automatisch aus Paket-Startdatum + Schicht-Nr. + Sperrmuster berechnet
SperrmusterDefiniert erlaubte Wochentage und Arbeitszeiten (z.B. Muster A: Mo–Fr, 23:00–05:00)

Mehrfachauswahl in der Schichtzuweisungs-Tabelle

Zeilen können per Checkbox ausgewählt werden (Shift-Klick für Bereichsauswahl). Änderungen an Baupaket, Schicht-Nr. oder Bedingungen werden für alle markierten Zeilen gleichzeitig übernommen.

Betriebliche Einschränkungen pro Schicht

Gleissperrung, Fahrstromunterbruch, Höhenbegrenzung und Nachbargleis-Sicherung können pro Standort separat gesetzt werden.

Auto-Zuweisung

Der Button Auto-Pakete verteilt alle Standorte automatisch auf Pakete und Schichten basierend auf Kapazität, Sperrmuster und Vorarbeiten. Manuelle Zuweisungen bleiben erhalten.

Fundamentebau-Gantt (pro Standort)

Jede Zeile entspricht einem Standort. Der Balken zeigt das zugewiesene Ausführungsdatum sowie den Aushärte-Zeitraum. Sortierung nach Datum, KM oder Baugruppe wählbar — Standardsortierung: Datum.

Baupaket erstellen und bearbeiten

+ Baupaket

Legt ein neues Paket an. Name, Farbe, Startdatum, Anzahl Nächte und Sperrmuster definieren.

Drag & Drop

Pakete können im Gantt direkt verschoben (Balken ziehen) oder verlängert/verkürzt werden (rechter Rand).

Abhängigkeiten

Pakete können als Nachfolger eines anderen Pakets definiert werden. Bei Verschiebung eines Vorgängers werden Nachfolger automatisch angepasst (Kaskade).

Sperrmuster

Definiert erlaubte Wochentage und Nettostunden. Das Muster steuert welche Kalendertage als Arbeitsnächte zählen.

Zoom-Stufen

Tag · Woche · Monat · Jahr — über die Zoom-Buttons oben rechts im Gantt wählbar.

Meilensteine

Über + Meilenstein können Termine (z.B. FL-Montage, Inbetriebnahme) als Rauten-Marker im Gantt eingetragen werden.

FT-Profile und Leistungsprofile

Der Tab Fundamente (in Phase Bauprojekt und Ausführung sichtbar) enthält die zentrale Bibliothek aller Fundamenttypen und Leistungsprofile des Projekts.

Fundamenttyp-Profile (FT)

Jeder Fundamenttyp definiert die geometrischen und technischen Eigenschaften eines Fundamenttyps:

FeldBeschreibung
Name / TypBezeichnung (z.B. «Blockfundament DP1a»)

Leistungsprofile definieren die Arbeitskapazität einer Maschine/Schicht. Sie werden Fundamenttypen zugewiesen und steuern die Auto-Zuweisung im Bauprogramm:

FeldBeschreibung

Standard-Fundamenttypen

Die App enthält vordefinierte Standard-FT (gemäss SBB-Fundamentliste). Diese können bearbeitet, aber nicht gelöscht werden. Eigene Spezial-FT können hinzugefügt werden.

FT-Zuweisung zu Standorten

In der Kachelansicht des Fundamente-Tabs können mehrere Standard-FT gleichzeitig ausgewählt und gemeinsam ein Leistungsprofil zugewiesen werden (Alle auswählen + Bulk-Zuweisung).

Datenspeicherung

Lokale Speicherung und Datensicherheit

Die App speichert alle Daten lokal im Browser (LocalStorage des Geräts). Es gibt keinen Server und keine Cloud-Anbindung.

Speicherort

LocalStorage

Alle Projektdaten, Einstellungen und Fotos werden im Browserspeicher des Geräts gehalten.

Kein Server

Daten verlassen das Gerät nur durch den manuellen JSON-Export.

Wichtige Hinweise

! Browser-Cache leeren löscht alle App-Daten unwiderruflich. Regelmässige JSON-Backups sind zwingend empfohlen.
i Auf mobilen Geräten kann der Browser Speicher freigeben, wenn der Systemspeicher knapp wird. Fotos als Base64 können viel Speicher belegen — regelmässig exportieren.

Speicherlimit

Browser erlauben typischerweise 5–50 MB LocalStorage. Bei sehr vielen hochauflösenden Fotos kann dieses Limit erreicht werden. Die App zeigt eine Warnung wenn der Speicher unter 10 % der verfügbaren Kapazität fällt.

Geräteübergreifend arbeiten

  1. Auf Gerät A: Backup exportierenEinstellungen → Backup (JSON) → Datei speichern.
  2. Datei übertragenPer E-Mail, USB oder Cloud-Dienst auf Gerät B übertragen.
  3. Auf Gerät B: Backup importierenEinstellungen → Backup wiederherstellen → Datei auswählen.
